스크립트 언어란

애플리케이션(프로그램)의 제어를 사용자의 요구에 맞게 수행하도록
매우 빠르게 배우고 짧은 소스코드로 상호작용 힐 수 있도록 고안된 언어들

자바스크립트

웹 브라우저상 가장 많이 기본적으로 사용하는 스크립트 언어
- 컴파일 => 컴파일 단게를 거치고 나서 실행되는 개념
- 인터프리터 방식 => 컴파일 단계를 거치지않고 곧바로 실행되는 개념
자바 / 자바스크립트 == 컴파일 + 인터프리터 방식(JIT Compiler)

  • 함수를 일급 객체로 취급
  • 문법 자체가 단순한 구조, 원칙을 가지고 있기 때문에 쉬움 예) 자바 : public, default, private 등의 접근제한자 개념 없음, 자바 : return ~ 등이 없고 함수 정의 시 반환형 정의 없음, 변수 선언 시 자료형을 지정하지 않음(타입추론)
  • 확장성이 좋아 배워두면 여기저기 쓸 수 있음

자바스크립트 소스코드 작성 위치에 따른 종류

  1. inline(인라인) : 요소 속성에 간단한 코드를 작성해서 실행
  2. internal(내부) : HTML문서 내에 script태그에 코드를 작성해서 실행
  3. external(외부) : 코드를 별도의 .js 파일로 작성해서 읽어오는 방법
    (모듈파일에서 export 사용하려는 쪽에서 import 해서 쓸 수 도 있음)

1) inline방식

        태그에 이벤트 속성에 실행할 간단한 소스코드를 작성하는 방식
        코드가 매~~~~~~~~~~~~~~~~~~~~~~~~~~~~우 소량이거나
        또는 다른 방식과 혼합해서 사용ㅎ야 할 때 고려해 볼 수 있음

        [표현법]
        <태그명 이벤트속성="실행하려고하는 소스코드">

2) internal 방식

        HTML문서에 script라는 태그 영역을 만든 뒤
        함수 단위로 소스코드를 작성하고
        특정 요소에 이벤트 발생 시 해당 함수를 호출하는 방식

        => script 태그 자체는 header, body 문서 내부 어디에다가 적어도 잘 동작하지만,
        브라우저 성능 향상을 위해 항상 스크립트 코드는 외부 방식을 사용하는 html요소 가장 하단에 연결시키는 것이 좋음
external 방식

별도의 .js파일로 소스코드를 작성하고 HTML문서에서 기져다 쓰는 방식
